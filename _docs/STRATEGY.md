# InterioRossi — Стратегия: Stitch + Claude Code

> Единый документ по workflow редизайна. Источник правды по тому, как
> Stitch и Claude Code работают вместе. Сверяться по нему; в новом чате —
> загрузить как контекст.

---

## 0. Принцип в одну фразу

**Stitch определяет, как должно выглядеть. Claude Code встраивает это в существующий сайт, ничего не ломая. Перевод между ними идёт через зафиксированную карту токенов — структура сайта при смене стиля не трогается.**

---

## 1. Ловушка, которую обходим

Неправильный путь: «генерю страницу в Stitch → экспортирую → вставляю в сайт → чиню в Claude Code». Ведёт к bug-by-bug охоте на каждой странице (белый текст, тёмные остатки, перебитая тема).

Корень — три несовместимости:
- Stitch пишет на **Tailwind + Material 3**, сайт — на **vanilla CSS с переменными**.
- Stitch генерит **тонкую структуру**, сайт — **богатую конверсионную** (квиз, тикер брендов, masonry-компетенции, 7-шаговый процесс, отзывы, FAQ).
- Stitch не знает про `main.js` (темы, карусели, формы).

**Вывод: Stitch не генератор страниц для прямой вставки.**

---

## 2. Роли инструментов

**Stitch — оракул стиля и прототипер компонентов.**
Его выход — спецификация, не код для вставки.
- Берём: палитра, шрифты, радиусы, отступы, тени, визуальные паттерны отдельных блоков.
- Не берём: готовый HTML целиком, Tailwind-классы, структуру страницы.

**Claude Code — инженер-исполнитель.**
Работает только с vanilla CSS, классами и темой сайта. Получает точечные инструкции. Не творит сам, не делает «красиво по своему вкусу».

**Чат с Claude (Эмиль ↔ Claude) — переводчик и архитектор.**
Берёт спеку из Stitch, переводит M3/Tailwind-токены в переменные сайта, решает reskin/редизайн поблочно, выдаёт Claude Code готовые промпты.

**Эмиль — арбитр.** Смотрит результат в браузере, говорит что не так.

## 4. Reskin vs Редизайн — правило по блоку

Один вопрос на каждый блок: **Stitch предлагает другой ВИД или другую СТРУКТУРУ?**

- **Вид** (цвет, шрифт, радиус, отступ, тень) → **reskin**, через тему/CSS, HTML не трогаем. ~90% случаев.
- **Структура** (bento вместо masonry, glass-карточка вместо карусели) → **осознанное решение**. Портируем только если объективно лучше конвертит или чище. Отдельная задача: HTML-правка + проверка `main.js`.

**Дефолт — reskin.** Структуру меняем только при сформулированном выигрыше конверсии. «Красивее» — не аргумент, если блок уже работает.

---

## 5. Theme-hardening — одноразовая инвестиция

Баги контраста случаются не из-за темы, а из-за того, что сайт не «theme-pure»: часть цветов зашита литералами (`#fff`, `#0c0f13`) мимо переменных.

**Действие (до тиража по страницам):** один системный прогон `style.css` — все литеральные цвета → переменные. После этого любой свап темы чистый.

Это то, что превращает набор фиксов в систему. Не косметика.

---

## 6. Пайплайн на одну страницу

1. **Stitch**: утверждаешь вид страницы. Экспорт → «Скопировать код» (токены блоков) + скриншот.
2. **Чат**: присылаешь скриншот + новые токены. Сверка с картой, решение reskin/редизайн поблочно.
3. **Чат → Claude Code**: точечные CSS-оверрайды (reskin) или хирургические HTML-правки (редизайн), всегда с явным «не трогай вот это».
4. **Claude Code**: применяет, репортит номера строк.
5. **Эмиль**: Live Server, скриншот сверху вниз.
6. Итерация до чистого результата → следующая страница.

Один цикл каждый раз. Без импровизации.

---

## 7. Гардрейлы

- Не вставлять Tailwind-разметку из Stitch в сайт — Tailwind там нет.
- Не давать Claude Code творческую свободу — только конкретные правки.
- Не трогать логику тем в `main.js` без захода через чат — она перебивает `data-theme`.
- Не править `style.css` ради темы — всё через `theme-cloud-dancer.css` (чистый свап и откат).
- Не менять структуру блока без причины конверсии.

## 8. Дисциплина версий

- Git вместо ручных `.BACKUP.css`.
- Ветка `redesign-cloud-dancer`, коммит после каждой зачищенной страницы.
- Откат поломки = одна команда.

---

## 9. Активация темы (как сейчас работает)

- Тема: `css/theme-cloud-dancer.css`, селектор `[data-theme="cloud-dancer"]`.
- Подключается в `<head>` ПОСЛЕ `style.css`.
- `<html lang="ru" data-theme="cloud-dancer">`.
- `initTheme()` в `main.js` ставит `cloud-dancer` дефолтом при пустом localStorage.
- Шрифты: EB Garamond (заголовки) + Manrope (текст) через Google Fonts.

## 10. Последовательность работ

1. Доделать главную (хедер + no-limits + тёмные остатки).
2. Theme-hardening (`style.css` → переменные).
3. Завести git.
4. Зафиксировать карту токенов (этот файл + `DESIGN_TOKENS_MAP.md`).
5. Страницы по пайплайну: catalog → about → contacts → materials → projects → partners.

Главная — эталон. Остальное — тираж.
